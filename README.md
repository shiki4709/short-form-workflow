# Short-Form Video Creation Workflow

A reusable step-by-step workflow for creating short-form educational/explainer videos (45-90s). Derived from the Sakana Fugu explainer video production process.

---

## Phase 1: Topic Selection & Research

**Goal:** Pick a topic, take a stance, verify facts.

### Steps
1. **Pick a trending topic** — scan news, X, Reddit for something your audience cares about
2. **Decide your angle** — are you explaining, reacting, comparing, or building? Pick one.
3. **Take a stance** — neutral, positive, critical, or "I tried it." Don't sit on the fence.
4. **Research the facts**
   - Read primary sources (official blog posts, papers, docs)
   - Verify specific claims (benchmark numbers, feature claims)
   - Flag what you're unsure about — don't script what you can't verify
5. **Write down the ONE core insight** — what's the single thing the viewer should walk away knowing?

### Output
- One-liner insight (e.g. "Fugu didn't build a smarter AI — they built a system that picks the smartest AI for you")
- List of verified facts with sources
- List of open questions that need resolving before scripting

---

## Phase 2: Script Writing

**Goal:** Write a speakable script in 50-60 seconds (70-110 words for 30s, ~150 words for 60s).

### Framework: Hook → Setup → Metaphor → Payoff → CTA

| Section | Duration | Purpose | Example |
|---------|----------|---------|---------|
| **HOOK** | 0-7s | Curiosity gap or bold claim. Viewer decides in 1.5s. | "This Japanese company claims their AI can compete with the strongest models. But nobody's explaining how." |
| **SETUP** | 7-15s | Establish the baseline so the metaphor makes sense | "Right now, when you use AI, you're going to one shop..." |
| **METAPHOR** | 15-35s | Core explanation using a relatable analogy | Boba shop → food court manager → catering planner |
| **PAYOFF** | 35-50s | So what? Why should the viewer care? | "What does this mean if you vibe code or use AI every day?" |
| **CTA** | 50-55s | Open-ended. Seed a follow-up or ask for engagement. | "I'm gonna try both and show you what happens. Follow if you want to see that." |

### Rules
- One idea per sentence
- Write for speech, not text (contractions, "you", 6th-grade reading level)
- Maximum 3 talking points per video
- Energy 20% higher than normal conversation
- Don't read a script — internalize it and speak naturally

### Iteration Process
1. **V1** — Get the structure down. Don't worry about word choice.
2. **V2** — Review with CriticMarkup comments. Flag weak spots.
3. **V3** — Resolve all comments. Final speakable version.

### Review Checklist (between drafts)
- [ ] Is the hook a curiosity gap or bold claim? (not a summary)
- [ ] Can you say each line out loud naturally?
- [ ] Is the metaphor self-contained? (viewer doesn't need prior knowledge)
- [ ] Are all facts verified?
- [ ] Is it under 60 seconds when spoken aloud?
- [ ] Does the CTA seed a follow-up or ask a question? (not "like and subscribe")

---

## Phase 3: Fact Verification

**Goal:** Ensure every claim in the script is accurate before filming.

### Steps
1. Cross-check numbers against primary sources
2. Use precise language ("competes with" vs "matches" vs "beats")
3. Remove competitor names if they create legal or relationship risk
4. If you can't verify a claim, cut it from the script

### Output
- Script with all claims verified and sourced
- Any caveats noted (e.g. "73.7 vs 80.3 — competes with, not matches")

## Phase 5: Film & Edit

**Goal:** Capture footage and edit into final video.

### Filming
- Talking head + b-roll
- Film hook separately — do multiple takes
- Speak naturally, don't read from script
- Energy 20% above normal

### Editing (CapCut)
- Add text overlays at every cut point
- Burn in subtitles (bilingual if applicable)
- Background music: consistent track across series
- Keep cuts tight — no dead air

### Specs
| Spec | Value |
|------|-------|
| Format | Vertical 9:16 |
| Resolution | 1080 x 1920 |
| Duration | 45-90 seconds |
| Subtitles | Burned-in |

## Phase 6: Post-Filming Audit

**Goal:** QA check before publishing. Based on June Yu Framework.

### 8-Point Checklist

| # | Criterion | What to Check |
|---|-----------|---------------|
| 1 | **Hook** | Does the first 3 seconds create a curiosity gap? Would YOU stop scrolling? |
| 2 | **Style** | Is the format consistent with the series? (talking head, split screen, etc.) |
| 3 | **Visuals & Edits** | Are overlays timed right? Any visible jump cuts that need masking? |
| 4 | **Cadence** | One idea per sentence? No rushed sections? |
| 5 | **Duration** | Under 60 seconds? (or under 90 for deeper content) |
| 6 | **Audio** | Clear voice? Music not competing with speech? |
| 7 | **Story Arc** | Does it build to a payoff? Is the ending OPEN, not concluded? |
| 8 | **CTA** | Does it invite action? (follow, comment, try it) |

### Quick Fixes

| Problem | Fix |
|---------|-----|
| Weak hook | Re-film first 3 seconds only |
| Too fast | Add 0.5s pauses between sections in edit |
| Visible cuts | Add b-roll or text overlay to mask |
| Flat ending | Re-record CTA with higher energy |
| Wrong music | Swap track, keep same vibe as series |

---

## Phase 7: Publish & Track

**Goal:** Post across platforms and track performance.

### Platforms
- YouTube Shorts
- Instagram Reels
- TikTok
- X (video post)

### Per-Video Tracking
- [ ] Posted on all platforms
- [ ] Captions/subtitles verified on each platform
- [ ] First 24h metrics noted (views, retention, comments)
- [ ] Learnings captured for next video
